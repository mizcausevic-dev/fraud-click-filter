# Origin

This repo exists because dirty traffic quietly damages commercial decision-making.

Most teams catch the problem too late:
- paid search looks productive because branded bot clicks inflate volume
- affiliate traffic gets credit without generating real humans
- retargeting looks efficient because synthetic sessions recycle impressions
- RevOps teams inherit polluted source data in the CRM

`fraud-click-filter` was designed as a Kinetic Gain portfolio build to show the missing bridge between edge protection and revenue truth.

The point is simple:
- block obvious bad traffic
- challenge uncertain traffic
- preserve legitimate traffic
- protect attribution before finance and growth reporting inherit bad data
